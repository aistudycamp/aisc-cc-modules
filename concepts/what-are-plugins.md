# Plugins

## What is it?

Plugins are like **apps you install from an app store**. Your phone comes with basic features, but when you want new capabilities -- a photo editor, a budgeting tool, a meditation timer -- you install an app someone else built. Plugins work the same way for Claude.

A plugin is the **packaging layer** for Claude Code extensions. It can bundle together skills, hooks, MCP server connections, and configurations into a single installable unit. Instead of building and configuring everything yourself from scratch, you install a plugin and immediately gain new abilities.

## Why does it matter?

Plugins save you enormous amounts of setup time. Rather than writing dozens of skills and configuring tools manually, you install a plugin and get a polished, tested toolkit in seconds. They also let you learn from how other people use Claude.

## How does it work?

- Plugins are shared packages that can contain skills, hooks, MCP servers, and configurations
- You browse available plugins in a marketplace or community directory
- Installing a plugin adds its files to your project's `.claude/` folder
- Once installed, the plugin's skills and capabilities are immediately available

## Real-world examples

- **A "superpowers" plugin:** Adds advanced planning, debugging, and code review workflows in one install
- **A content-creation plugin:** Bundles skills for blog writing, social media, newsletters, and SEO optimization
- **A data-analysis plugin:** Provides skills for cleaning data, generating charts, and writing summary reports
- **A frontend-design plugin:** Includes skills for building polished web interfaces with modern design patterns
