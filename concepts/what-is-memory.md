# Memory

## What is it?

Memory is like a **notebook Claude keeps between conversations**. Normally, when you close a chat with an AI and start a new one, it forgets everything. Memory changes that. It lets Claude write down important things and read them back later, so it picks up where you left off.

When Claude learns something important during a session -- your preferences, a decision you made, context about your project -- it can save that as a memory. The next time you start a conversation, Claude checks its notebook and already knows what's going on.

## Why does it matter?

Memory means you stop repeating yourself. Instead of re-explaining your project, your preferences, or past decisions every session, Claude remembers them. This makes each conversation faster and more useful than the last.

## How does it work?

- During a conversation, Claude can save important information as memories
- Memories are stored as small text files on your machine, inside your project
- When a new session starts, Claude reads these memories to restore context
- You can review, edit, or delete memories anytime -- you stay in control

## Real-world examples

- **Remembering preferences:** "Nicole prefers bullet points over long paragraphs" carries across every future session
- **Tracking project context:** "We decided to use blue as the primary brand color at the March 15 meeting" stays available without re-explaining
- **Recalling past decisions:** "We chose Mailchimp over ConvertKit for the newsletter -- here's why" so you don't revisit settled debates
- **Keeping a running log:** "Sprint 1 focused on the landing page; Sprint 2 is tackling the checkout flow" so Claude always knows where the project stands
